# cfnat 本地订阅生成器 GUI

基于 `tkinter` 的图形界面版本，用来监控 `cfnat` 输出、提取有效 IP、生成本地订阅文件，并提供本地 HTTP 订阅地址。

本公开仓库只保留 GUI 脚本、说明文档和公开可执行文件，不包含测试脚本、历史开发文档或其它私有资料。

## 功能特性

### 核心功能

- 实时监控 `cfnat` 输出，自动提取有效 IP
- 自动替换模板中目标节点的 IP 地址
- 自动生成本地订阅文件 `subscription.txt`
- 提供本地 HTTP 订阅服务，默认地址 `http://127.0.0.1:8888/sub`
- 支持历史订阅恢复，重启后可继续提供订阅
- 支持手动 IP 启动

### 扫描与订阅逻辑

- 默认延迟阈值为 `300ms`
- 默认按扫描结果里有效 IP 数量最多的机房给出建议
- 订阅 IP 达到稳定刷新次数后再切换，减少频繁波动
- 发现异常切换或 IP 库耗尽时，会给出提示

### 日志与界面

- 实时显示扫描进度、IP 刷新、订阅更新等信息
- 关键刷新信息按单行滚动更新，减少刷屏
- 普通结果日志会保留，便于回看
- 支持可选的本地日志保存

## 当前订阅测速

GUI 中提供了“测速当前订阅”按钮。

用途：

- 对当前订阅 IP 做一次独立测速
- 首次运行但已有历史 `subscription.txt` 时，也可以直接测速历史订阅 IP
- 测速结果会保留在日志区，不会被后续 `IP刷新` 行覆盖

测速口径：

- 固定使用 `10MB` 下载文件
- 结果显示为 `MB/s`
- 更适合做不同 IP 之间的相对比较，不等同于标准宽带测速

## 测速原理

为避免测到代理出口速度，测速不会走系统代理或环境变量代理。

实现方式：

- 使用 Python 标准库 `socket + ssl`
- 直接连接当前订阅对应的目标 IP 和端口
- HTTPS 场景下保留目标站点的 `Host/SNI`
- 按时间窗口统计下载速度并输出结果

这意味着测速结果更接近“当前订阅 IP 直连测速目标时的可用吞吐”，而不是本机代理链路的最终速度。

## 快速开始

### 前置要求

- Windows
- Python 3.6+
- 可用的 `tkinter`
- `cfnat` 可执行文件
- 节点模板文件 `nodes.txt`

### 启动方式

双击：

```text
cfnat_sub_gui.pyw
```

或命令行：

```bash
python cfnat_sub_gui.pyw
```

### 使用流程

1. 准备好 `nodes.txt` 节点模板。
2. 启动 `cfnat_sub_gui.pyw`。
3. 调整机房、延迟、并发等参数。
4. 点击“启动扫描”开始优选。
5. 将本地订阅地址填入客户端使用。
6. 如需对当前订阅 IP 做对比，点击“测速当前订阅”。

## 节点模板

`nodes.txt` 支持常见公开格式，例如：

- `vless://...`
- `vmess://...`

程序会按当前优选结果替换目标节点地址，并生成新的订阅内容。

## 输出文件

运行过程中可能生成或更新：

- `subscription.txt`
- `cfnat_sub.pid`
- `logs/` 下的日志文件
- 位置缓存和统计文件

这些都是本地运行产物，不属于公开仓库必须提交的内容。
