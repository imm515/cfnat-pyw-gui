# cfnat 本地订阅生成器 GUI

基于 `tkinter` 的 Windows GUI 工具，用来监控 `cfnat` 输出、维护本地订阅文件，并提供本地 HTTP 订阅地址。

这个公开仓库只保留 GUI 脚本、公开说明和可执行文件，不包含测试脚本、历史开发文档或其它私有工作文件。

## 功能

- 实时读取 `cfnat` 输出，提取有效 IP
- 自动更新本地 `subscription.txt`
- 提供本地订阅地址，默认 `http://127.0.0.1:8888/sub`
- 支持历史订阅恢复，重启后可继续提供订阅
- 支持手动 IP 启动
- 支持当前订阅测速
- 支持日志显示与可选日志保存
- 默认延迟阈值为 `300ms`
- 默认按扫描结果里 IP 数量最多的机房给出建议

## 当前订阅测速

GUI 中提供了“测速当前订阅”按钮。

用途：

- 对当前正在使用的订阅 IP 做一次独立测速
- 首次运行但存在历史 `subscription.txt` 时，也可以直接测速历史订阅 IP
- 测速结果会作为普通日志保留，不会被后续 `IP刷新` 行覆盖

测速口径：

- 固定使用 `10MB` 下载文件
- 默认目标为 `speed.cloudflare.com/__down?bytes=10000000`
- 按时间窗口统计瞬时速度，结果显示为 `MB/s`
- 适合做相对比较，不等同于标准宽带测速

## 测速原理

为避免测到代理出口速度，测速不会走系统代理或环境代理。

实现方式：

- 直接使用 Python 标准库 `socket + ssl`
- 直接连接当前订阅对应的目标 IP 和端口
- HTTPS 场景下保留目标站点的 `Host/SNI`
- 不引入第三方测速库，尽量兼容 Win7 场景

这意味着测速结果更接近“当前订阅 IP 直连该目标时的可用吞吐”，而不是本机代理链路的最终速度。

## 使用方式

### 1. 准备文件

目录中至少需要：

- `cfnat_sub_gui.pyw`
- `cfdata-cli-win7.exe` 或可用的 `cfnat` 可执行文件
- `nodes.txt` 节点模板

### 2. 启动

双击：

```text
cfnat_sub_gui.pyw
```

或命令行：

```bash
python cfnat_sub_gui.pyw
```

### 3. 扫描与订阅

- 点击“启动扫描”开始优选
- 程序会持续更新当前 IP、订阅 IP 和本地订阅文件
- 本地订阅地址默认是：

```text
http://127.0.0.1:8888/sub
```

### 4. 测速

- 当订阅 IP 已经生成后，点击“测速当前订阅”
- 日志中会显示开始测速、测速方式和最终速度结果

## 节点模板

`nodes.txt` 支持常见公开格式，例如：

- `vless://...`
- `vmess://...`

程序会按当前优选到的 IP 替换模板中对应目标节点的地址，并生成新的订阅内容。

## 适用环境

- Windows
- Python 3.6+
- 带 `tkinter` 的 Python 运行环境

Win7 兼容策略：

- 不依赖第三方库
- GUI 和测速逻辑均使用 Python 标准库实现
- 适合在较旧环境下直接运行

## 输出文件

运行过程中可能生成或更新：

- `subscription.txt`
- `cfnat_sub.pid`
- `logs/` 下的日志文件
- 位置缓存和统计文件

这些都是本地运行产物，不属于公开仓库必须内容。

## 说明

- 本项目更适合用于 IP 相对优选和订阅维护
- 测速结果会受到目标 IP、网络时段、线路波动影响
- 若需要长期稳定，建议结合你的实际客户端环境持续观察
